#### 1. 상품

|------------------------------------------------------------------------------------------------------|-------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------|

| 사용자는 검색어(상품 이름), 가격 범위, 카테고리 설정을 통해 상품을 필터링, 검색할 수 있다.          | 시스템에 검색 가능한 상품 정보가 존재한다. | 1. 사용자가 검색어, 가격 범위, 카테고리 등을 입력하여 상품 검색 요청을 보낸다.<br>2. 시스템이 입력된 조건에 맞는 상품을 데이터베이스에서 검색한다.<br>3. 시스템이 검색 결과를 사용자에게 반환한다.<br>4. 사용자가 검색 결과를 확인한다.            | 사용자는 검색 조건에 맞는 상품 리스트를 확인할 수 있다. |
| 사용자는 상품 리스트 중 특정 상품을 선택해 특정 상품의 이름, 이미지, 상품 설명, 가격 정보, 남은 수량을 확인할 수 있다. | 시스템에 상품 상세 정보가 존재한다. | 1. 사용자가 특정 상품을 선택하여 상세 정보 조회 요청을 보낸다.<br>2. 시스템이 해당 상품의 상세 정보를 데이터베이스에서 가져온다.<br>3. 시스템이 사용자에게 상품의 상세 정보를 반환한다.<br>4. 사용자가 상품의 상세 정보를 확인한다.          | 사용자는 특정 상품의 상세 정보를 확인할 수 있다. |
| 사용자는 특정 상품의 갯수를 선택해 장바구니에 담거나, 바로 결제할 수 있다.                         | 사용자가 로그인 상태여야 한다. | 1. 사용자가 특정 상품의 갯수를 선택한다.<br>2. 사용자가 장바구니에 담기 또는 바로 결제하기 버튼을 클릭한다.<br>3. 시스템이 장바구니에 담기 요청일 경우 해당 상품을 장바구니에 추가한다.<br>4. 바로 결제하기 요청일 경우 결제 페이지로 이동한다.<br>5. (옵션) 사용자가 결제 정보를 입력하고 결제 요청을 보낸다. | 사용자는 선택한 상품을 장바구니에 담거나 결제 절차를 진행할 수 있다. |
| 상품은 영화, 책, 앨범 3종류로 가정하며, 각 상품은 세부 카테고리 정보를 가진다.                    | 시스템에 영화, 책, 앨범 정보가 존재한다. | 1. 사용자가 특정 카테고리를 선택하여 상품 리스트를 조회 요청한다.<br>2. 시스템이 선택된 카테고리에 해당하는 상품 정보를 데이터베이스에서 가져온다.<br>3. 시스템이 사용자에게 해당 카테고리의 상품 리스트를 반환한다.<br>4. 사용자가 반환된 상품 리스트를 확인한다. | 사용자는 선택된 카테고리의 상품 리스트를 확인할 수 있다. |

#### 2. 사용자

| 기능                                                                 | 사전 조건                    | 이벤트 흐름                                                                                                                                                                                                                                                                                                | 사후 조건                               | 예외/제약사항                                                                                                                                                                                |
|----------------------------------------------------------------------|-----------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 사용자는 아이디, 비밀번호, 주소, 이메일 정보를 입력해 회원 가입이 가능하다. | 가입하지 않은 유저여야 한다. | 1. 사용자가 회원가입 양식을 작성하여 제출한다.<br>2. 시스템이 입력된 정보를 검증한다.<br>3. 시스템이 아이디 또는 이메일 중복 여부를 확인한다.<br>4. 중복이 없고 비밀번호 조건이 충족되면 회원 정보를 데이터베이스에 저장한다.<br>5. 회원가입 성공 메시지를 사용자에게 반환한다.                     | 새로운 사용자 계정이 생성되고, 사용자는 로그인할 수 있다. | Exception 1: 아이디 또는 이메일 정보가 이미 가입되어 있는 유저의 것과 동일하다면 서비스는 회원가입을 거부한다.<br>Exception 2: 비밀번호는 영문, 숫자, 특수문자를 모두 포함하며, 8자 이상이어야 한다. 이 조건을 만족하지 못한다면 서비스는 회원가입을 거부한다. |
| 사용자는 Kakao, Google을 사용한 OAuth2 로그인/ 회원가입이 가능하다. | OAuth2 제공자와의 연동이 설정되어 있어야 한다. | 1. 사용자가 Kakao 또는 Google 로그인 버튼을 클릭한다.<br>2. 시스템이 해당 OAuth2 제공자의 인증 페이지로 리디렉션한다.<br>3. 사용자가 OAuth2 인증을 완료한다.<br>4. 시스템이 OAuth2 제공자로부터 사용자 정보를 받아온다.<br>5. 해당 이메일이 이미 존재하는지 확인한다.<br>6. 존재하면 계정을 통합하고, 존재하지 않으면 추가 정보(주소 등)를 입력받는다.<br>7. 회원 정보를 데이터베이스에 저장하고 로그인 처리한다. | 사용자는 OAuth2를 통해 로그인 및 회원가입을 완료할 수 있다. | 제약사항 1: 사용자가 처음 OAuth2로 로그인 했으나 이미 해당 이메일이 가입되어 있다면, 두 계정을 통합한다.<br>제약사항 2: 사용자가 처음 OAuth2로 로그인하는 경우 해당 이메일이 존재하지 않는다면, 시스템은 주소 정보를 추가로 입력받아 저장한다. |
| 사용자는 자신의 이름, 이메일, 주소, 프로필 이미지 정보를 조회할 수 있다. | 사용자가 로그인 상태여야 한다. | 1. 사용자가 자신의 프로필 조회 요청을 보낸다.<br>2. 시스템이 사용자 정보를 데이터베이스에서 가져온다.<br>3. 시스템이 사용자에게 이름, 이메일, 주소, 프로필 이미지 정보를 반환한다.<br>4. 사용자가 반환된 정보를 확인한다.                                                                      | 사용자는 자신의 프로필 정보를 확인할 수 있다. |                                                                                                                                                                                              |
| 사용자는 자신의 이름, 주소, 프로필 이미지를 수정할 수 있다.         | 사용자가 로그인 상태여야 한다. | 1. 사용자가 수정할 프로필 정보를 입력하고 수정 요청을 보낸다.<br>2. 시스템이 입력된 정보를 검증한다.<br>3. 시스템이 사용자 정보를 데이터베이스에 업데이트한다.<br>4. 시스템이 수정 완료 메시지를 사용자에게 반환한다.                                                                              | 사용자의 프로필 정보가 업데이트된다.  |                                                                                                                                                                                              |
| 사용자는 자신의 비밀번호를 수정할 수 있다. 이 때 변경되기 이전의 비밀번호를 인증을 위해 추가로 입력해야 한다. | 사용자가 로그인 상태여야 한다. | 1. 사용자가 현재 비밀번호와 새로운 비밀번호를 입력하여 비밀번호 수정 요청을 보낸다.<br>2. 시스템이 현재 비밀번호를 검증한다.<br>3. 새로운 비밀번호가 조건을 만족하는지 검증한다.<br>4. 조건이 만족되면 비밀번호를 업데이트한다.<br>5. 수정 완료 메시지를 사용자에게 반환한다.                           | 사용자의 비밀번호가 업데이트된다.      |                                                                                                                                                                                              |
| 사용자는 자신의 가상 계좌의 이름을 설정해 가상계좌를 추가할 수 있다. | 사용자가 로그인 상태여야 한다. | 1. 사용자가 가상 계좌 이름을 입력하여 가상 계좌 추가 요청을 보낸다.<br>2. 시스템이 가상 계좌 정보를 생성하여 데이터베이스에 저장한다.<br>3. 추가 완료 메시지를 사용자에게 반환한다.                                                                                                             | 사용자의 가상 계좌가 추가된다.          |                                                                                                                                                                                              |
| 사용자는 자신의 가상 계좌 리스트를 조회할 수 있다.                 | 사용자가 로그인 상태여야 한다. | 1. 사용자가 가상 계좌 리스트 조회 요청을 보낸다.<br>2. 시스템이 사용자 가상 계좌 정보를 데이터베이스에서 가져온다.<br>3. 시스템이 가상 계좌 리스트를 사용자에게 반환한다.<br>4. 사용자가 반환된 가상 계좌 리스트를 확인한다.                                                                  | 사용자는 자신의 가상 계좌 리스트를 확인할 수 있다. |                                                                                                                                                                                              |
| 사용자는 자신의 가상 계좌에 금액을 충전할 수 있다.                 | 사용자가 로그인 상태여야 하며, 충전할 금액을 입력할 수 있어야 한다. | 1. 사용자가 충전할 금액을 입력하여 가상 계좌 충전 요청을 보낸다.<br>2. 시스템

이 충전할 금액을 검증한다.<br>3. 시스템이 가상 계좌에 금액을 추가한다.<br>4. 충전 완료 메시지를 사용자에게 반환한다.                                                                                           | 사용자의 가상 계좌에 금액이 충전된다.  |                                                                                                                                                                                              |
| 사용자는 자신의 가상계좌가 아닌 가상계좌에는 금액을 충전할 수 없다. | 사용자가 로그인 상태여야 한다. | 1. 사용자가 다른 사용자의 가상 계좌에 금액 충전 요청을 보낸다.<br>2. 시스템이 가상 계좌 소유자를 확인하고 요청을 거부한다.<br>3. 충전 실패 메시지를 사용자에게 반환한다.                                                                                                             | 다른 사용자의 가상 계좌에는 금액이 충전되지 않는다. |                                                                                                                                                                                              |

#### 3. 장바구니 관리

| 기능                                                                                             | 사전 조건                        | 이벤트 흐름                                                                                                                                                                                                                                           | 사후 조건                                    |
|--------------------------------------------------------------------------------------------------|---------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------|
| 사용자는 특정 상품과 갯수를 선택해 장바구니에 담을 수 있다.                                     | 사용자가 로그인 상태여야 한다.   | 1. 사용자가 특정 상품과 갯수를 선택하여 장바구니에 담기 요청을 보낸다.<br>2. 시스템이 장바구니에 해당 상품과 갯수를 추가한다.<br>3. 추가 완료 메시지를 사용자에게 반환한다.                                                                                  | 사용자의 장바구니에 선택한 상품이 추가된다.  |
| 사용자는 장바구니 리스트에서 특정 물건의 갯수를 조절할 수 있다.                                 | 사용자가 로그인 상태여야 하며, 장바구니에 상품이 담겨 있어야 한다. | 1. 사용자가 장바구니 리스트에서 특정 상품의 갯수를 변경하여 수정 요청을 보낸다.<br>2. 시스템이 장바구니에 해당 상품의 갯수를 업데이트한다.<br>3. 수정 완료 메시지를 사용자에게 반환한다.                                                                     | 사용자의 장바구니에 담긴 상품의 갯수가 업데이트된다. |
| 사용자는 장바구니 리스트에서 특정 물건을 선택해 주문할 수 있다. 시스템은 결제 완료된 물건은 장바구니에서 제거한다. | 사용자가 로그인 상태여야 하며, 장바구니에 주문 가능한 상품이 담겨 있어야 한다. | 1. 사용자가 장바구니 리스트에서 특정 상품을 선택하여 주문 요청을 보낸다.<br>2. 시스템이 선택된 상품의 재고를 확인하고 결제 페이지로 이동한다.<br>3. 사용자가 결제를 완료한다.<br>4. 시스템이 결제 완료된 상품을 장바구니에서 제거한다.<br>5. 주문 완료 메시지를 사용자에게 반환한다. | 사용자의 장바구니에서 결제 완료된 상품이 제거되고, 주문이 완료된다. |

#### 4. 주문
